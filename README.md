# Lexical Analyser

A small compiler-design project built with Lex to tokenize input source code and classify common language elements such as keywords, identifiers, literals, operators, and punctuation.

## Why this repository exists

This repository is an early systems-oriented learning project that I keep as a historical artifact. It reflects hands-on work with lexical analysis and the mechanics of building a tokenization pipeline rather than a polished production tool.

## What it demonstrates

- pattern-based token matching with Lex
- classification of keywords, identifiers, literals, and operators
- simple handling of comments and output generation
- early exposure to compiler-design concepts through implementation

## Repository structure

The core files are:

- `Lexical Analyser/lexi.l`: lexical rules and token classification logic
- `Lexical Analyser/abc.c`: sample input program
- `Lexical Analyser/tokens.txt`: generated token output sample

## What I owned

I built this as a compiler-design exercise to understand how token recognition works in practice and how a lexer can be assembled from rule definitions and supporting C code.

## Public positioning

This repository is preserved to show early low-level systems coursework. It should be read as a learning artifact and not as a representation of my current production engineering style.
